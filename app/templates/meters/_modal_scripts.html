{# Zentrale JS-Helfer fuer die drei Meter-Modale: - Reading-Edit-Modal (Zaehlerstand bearbeiten/anlegen) → openReadingModal - Meter-Replace-Modal (Zaehlerwechsel) → openReplaceModal - Meter-Edit-Modal (Zaehler-Stammdaten bearbeiten) → openMeterEditModal Per Inkludieren im {% block scripts %} der jeweiligen Seite verfuegbar machen -- erwartet, dass die zugehoerigen Modal-Partials (eines oder mehrere) in {% block modals %} eingebunden sind. Fehlende Modale werden uebersprungen. Refresh-Strategie: - Die Routes (POST mit `X-From-Modal: 1`) liefern 204 + HX-Trigger-Events (`readingSaved` / `meterReplaced` / `meterEdited`) zurueck -- der Server entscheidet NICHT, wie die aufrufende Seite zu refreshen ist. - Jede Seite kann ihren eigenen Handler definieren: window.onReadingSaved(e) / onMeterReplaced(e) / onMeterEdited(e). Default (kein Handler gesetzt): window.location.reload(). #}
